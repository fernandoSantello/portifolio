    Sou um desenvolvedor experiente aprendendo mais sobre as áreas que me apaixonei. Com muito esforço, prática e experiência, sei que vou conseguir alcançar meu objetivo de ter uma carreira cheia de problemas pra resolver e aprender com eles, nunca parando de crescer.

    Outra coisa sobre mim, é que eu sou o Dev comunicativo. Gosto muito de falar com pessoas, entender os problemas delas e o que fazem elas sentirem que aquilo são problemas (quase fiz faculdade de psicologia). Sou muito bom em me relacionar e isso tem me ajudado para caramba na hora de transmitir os resultados e entender mais profundamente os problemas que elas encontram no dia a dia.

    Desenvolvimento Backend

    Python

    Python

    Python é uma linguagem de programação de alto nível muito popular. Rápido de codar e rápido de implementar soluções que já comecem a agregar valor para a empresa.

    É a linguagem que mais utilizei no âmbito profissional,implementando diferentes tipos de soluções com ela. Isso engloba muitas coisas. RPAs que realizam busca de informação na web usando Selenium ou Playwright, que geram documentos em massa em sites do governo e que realizam qualquer processo que uma pessoa faz. Nesse pacote também está incluso APIs, seja interagindo com disponibilizadas por diferentes empresas ou desenvolvendo as próprias usando Flask como base, servindo como middleware para muitos outros programas desenvolvidos.

    Também utilizei Python profissionalmente para criação de fluxos ETL com informações de bases on-premisse ou outputs de programas. Também cheguei a dar uma explorada na parte de testagem de software recentemente, que funciona muito bem.

    E não para por aí! Ele tambéme está sendo meu companheiro para escrever funções AWS Lambda e para engenharia de dados. Python é muito amplo e sempre dá para aprender mais e melhorar as habilidades.

    Resumindo: adoro Python e tenho um adesivo no meu computador.

    SQL

    ×

    SQL

    SQL é uma linguagem específica para gerenciamento e consulta de bancos de dadose e foi minha introdução ao mundo dos dados. Durante minha faculdade tive um professor de banco de dados fenomenal, que ajudou muito a criar mais curiosidade sobre o tópico. Sempre me esforácava ao máximo para criar os melhores trabalhos, criando estruturas relacionais íntegras e coesas utilziando o SQL Server da Microsoft. Meu trabalho de conclusão de curso teve como tema "Automação do Processo de Normalização de Banco de Dados" e agora ele está na biblioteca da UNIP, já que obteu a nota máxima.

    Posteriormente no trabalho utilizei muito SQL para realizar consultas e extrair informações de bases on-premisse antigas, para tirar insigth dê informações. Realizar esse processo em uma base desconhecida e não documentada é algo que leva tempo, mas a sensação de relacionar as tabelas e extrair informações de valor sempre é maravilhosa.

    Todos os bancos de dados desenvolvido para as aplicações que tenho em produção utilizam MySQL como banco de dados principal, então adquiri muita experiência no processo de arquitetura e modelagem de dados. Sei o processo de ponta a ponta e adoro documentar para que tudo fique extremamente organizado e de fácil entendimento.

    Power BI

    ×

    Power BI

    Power BI faz parte da Power Platform e é a ferramenta de Business Inteligence e Data Visualization do pacote.

    Tendo trabalhado muito com dados, eu precisei não só extrair e armazenar as informações de maneira segura, mas também tirar insigths dos dados e criar visualizações coerentes para mostrar a times e equipe de diretoria. Não basta ter informações com valor, é necessário saber como transmitir esse valor para tomadores de decisão que vão fazer a diferença. Fazer a conexão das várias partes do PowerBI com bancos on-premisse é algo que sei bem, da mesma forma como sei como extrair informações que estão sendo armazenadas no Dataverse.

    Aprender Power BI (e me apronfudar nos estudos durante a PL900) é algo que me ajuda muito na visualização de todo o ciclo de vida dos dados, desde a geração/extração deles em algum lugar, até a visualização desses dados para pessoas chave, incentivando a criação de uma cultura Data Driven.

    Power Apps

    Power Apps é um conjunto de aplicativos, serviços e conectores, bem como uma plataforma de dados que oferece um ambiente de desenvolvimento rápido de aplicativos para criação dos apps personalizados para suas necessidades de negócios. Sua organização utiliza Microsoft e está precisando de uma solução bem estruturada? Precisa desenvolver um backend e um frontend de uma aplicação que será utilizada semana que vem em pouquíssimo tempo? Power Apps é um componente dos ingredientes para isso.

    Minha prática com Power Apps começou em minha primeira experiência profissional e segue até hoje. Desenvolver soluções de maneira veloz que o usuário final pode usar é fantástico, começa-se a agragar valor muito rápido. Tenho experiência com canvas-apps e model-driven apps, fazendo aplicativos de agendamento, manutenção de estoque, fluxo de processo de aprovação, entre outros.

    Power Automate

    Power Automate

    Power Automate é uma plataforma SaaS da Microsoft para automação de tarefas recorrentes. Essas tarefas podem ter diferentes tipos de gatilho e seguir diferentes caminhos, mas a ideia de automação de algum processo recorrente segue a mesma.

    Minhasexperiência com Power Automate é bem extensa, fazendo fluxos simples de mensagens/emails automatizados e processos de aprovação, até fluxos entre diferentes Business Units para medição do valor químico de um produto. Amo toda abertura que tenho para automatizar processos de forma fácil e eficiente.

    Docker

    Docker

    Docker é um conjunto de produtos de plataforma como serviço que usam virtualização de nível de sistema operacional para entregar software em pacotes chamados contêineres. Os contêineres são isolados uns dos outros e agrupam seus próprios softwares, bibliotecas e arquivos de configuração.

    Como todo desenvolvedor, logo percebi que o meu código nem sempre funcionava no computador de meus colegas de trabalho, e vice-versa. Para criarmos a solução para este problema, logo tivemos projetos profissionais colocados em um on-premisses registry e em contêineres, fazendo com que as aplicações funcionassem em qualquer máquina.
